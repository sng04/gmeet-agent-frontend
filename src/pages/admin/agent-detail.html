What's the best way to set up a multi-region failover?
Based on our Cloud Migration Playbook (Section 4.2), the recommended approach for multi-region failover is to use Route 53 health checks with active-passive configuration. The primary region handles all traffic, and Route 53 automatically fails over to the secondary region if health checks fail for 3 consecutive intervals.
